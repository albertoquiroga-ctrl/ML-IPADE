Driver Churn — Análisis completo¶

Caso: Unexpected expectation: Why do drivers churn in a transportation platform? (IPADE, Feb 2023)

Este notebook ejecuta el plan de análisis acordado, en 7 fases:

  1. Limpieza y preparación
  2. EDA orientado a hipótesis
  3. Modelo logístico v1 (todas las variables, según pide el caso)
  4. Modelo v2 con ingeniería de variables (brecha salarial)
  5. Análisis de cutoff: 0.15 vs 0.30
  6. Resumen de coeficientes para campaña de retención
  7. Limitaciones

Cada celda imprime resultados clave en texto plano para que sean fáciles de compartir y discutir.

df = raw.copy()

# 1) Bajar nombres a minúsculas
df.columns = [c.strip().lower() for c in df.columns]

# 2) Eliminar columnas fantasma totalmente vacías
empty_cols = [c for c in df.columns if df[c].isna().all()]
if empty_cols:
    df = df.drop(columns=empty_cols)
    print("Columnas vacías eliminadas:", empty_cols)

# 3) Limpiar salarios: vienen como strings tipo ' 21,991 '
def to_number(s):
    if pd.isna(s):
        return np.nan
    return float(str(s).replace(",", "").replace(" ", ""))

for col in ["expsal", "actsal"]:
    df[col] = df[col].apply(to_number)

# 4) Normalizar categóricas
df["status"]   = df["status"].str.strip().str.upper()
df["children"] = df["children"].str.strip().str.capitalize()

print("\nDtypes después de limpieza:")
print(df.dtypes)

# 2.3 — Hipótesis clave: la BRECHA salarial
# SHORTFALL_PCT > 0  =>  el driver gana MENOS de lo que esperaba
df['salgap']        = df['expsal'] - df['actsal']
df['shortfall_pct'] = (df['expsal'] - df['actsal']) / df['expsal']

print("Brecha promedio (USD) por churn:")
print(df.groupby('churn')[['salgap', 'shortfall_pct']].mean())

# Tasa de churn por decil de shortfall_pct
df['shortfall_decile'] = pd.qcut(df['shortfall_pct'], 10, labels=False, duplicates='drop')
churn_by_decile = df.groupby('shortfall_decile')['churn'].agg(['mean', 'count'])
print("\nTasa de churn por decil de shortfall_pct (decil 0 = gana más de lo esperado, 9 = gana mucho menos):")

Fase 3 — Modelo logístico v1: todas las variables¶

Esto es lo que pide explícitamente la pregunta 1 del caso.

Especificación:

$$\text{logit}(\Pr(\text{churn}=1)) = \beta_0 + \beta_1 \text{age} + \beta_2 \text{status\_single} + \beta_3 \text{children\_yes} + \beta_4 \text{expsal} + \beta_5 \text{actsal}$$

Usamos statsmodels (no solo sklearn) para obtener errores estándar y p-values, que son indispensables para interpretar el modelo en términos de negocio.

Fase 4 — Modelo v2 con ingeniería de variables¶

Hipótesis a contrastar: la brecha entre lo esperado y lo real (shortfall_pct) explica el churn mejor que expsal y actsal por separado, y elimina la multicolinealidad detectada en v1.

Especificación v2:

$$\text{logit}(\Pr(\text{churn}=1)) = \beta_0 + \beta_1 \text{age} + \beta_2 \text{status\_single} + \beta_3 \text{children\_yes} + \beta_4 \text{shortfall\_pct}$$

def evaluate_cutoff(y_true, y_proba, cutoff):
    y_hat = (y_proba >= cutoff).astype(int)
    tn, fp, fn, tp = confusion_matrix(y_true, y_hat).ravel()
    precision = tp / (tp + fp) if (tp + fp) > 0 else 0.0
    recall    = tp / (tp + fn) if (tp + fn) > 0 else 0.0
    f1        = 2 * precision * recall / (precision + recall) if (precision + recall) > 0 else 0.0
    return dict(cutoff=cutoff, TP=tp, FP=fp, FN=fn, TN=tn,
                precision=precision, recall=recall, f1=f1, flagged=tp + fp)

# Costo total esperado por cutoff
# Lógica:
#   - Falso negativo (FN): perdemos un driver que se va sin que lo detectemos -> COST_PER_FALSE_NEGATIVE_USD
#   - Falso positivo (FP): contactamos a alguien que no se iba a ir -> COST_PER_FALSE_POSITIVE_USD
#   - Verdadero positivo (TP): contactamos correctamente.
#       * Pagamos COST_PER_FALSE_POSITIVE_USD por la intervención.
#       * Recuperamos (RETENTION_SUCCESS_RATE * COST_PER_FALSE_NEGATIVE_USD) en valor evitado.
#   - Verdadero negativo (TN): cero costo.

def expected_cost(row):
    fn_cost = row['FN'] * COST_PER_FALSE_NEGATIVE_USD
    fp_cost = row['FP'] * COST_PER_FALSE_POSITIVE_USD
    tp_intervention_cost   = row['TP'] * COST_PER_FALSE_POSITIVE_USD
    tp_value_recovered     = row['TP'] * RETENTION_SUCCESS_RATE * COST_PER_FALSE_NEGATIVE_USD
    total = fn_cost + fp_cost + tp_intervention_cost - tp_value_recovered
    return total

res['expected_total_cost_usd'] = res.apply(expected_cost, axis=1)

Fase 7 — Limitaciones (ser explícito antes de que las pregunten los VCs)¶

  • Datos point-in-time: no podemos modelar cuándo va a churnar el driver, solo si lo hará. Para un modelo de supervivencia haría falta historial.
  • Variables ausentes: antigüedad en la plataforma, frecuencia de viajes, rating del conductor, zona geográfica, tasa de cancelación. Cualquier modelo serio en producción necesita estas variables. Lo más probable es que mejoren AUC sustancialmente.
  • Tamaño de muestra: 1,046 observaciones es razonable para un modelo paramétrico, pero limita la cantidad de interacciones que podemos estimar de forma estable.
  • Sesgo de selección: la encuesta capturó drivers con cuenta activa o reciente. Drivers que churnearon hace mucho pueden estar subrepresentados.
  • No causalidad: el modelo describe asociaciones, no efectos causales. Para validar cualquier intervención hay que correr A/B test.
  • Estabilidad temporal: los coeficientes pueden cambiar con el tiempo (por ejemplo, si la plataforma sube comisiones). Hay que re-entrenar periódicamente.
